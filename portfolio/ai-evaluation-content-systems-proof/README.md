# AI evaluation content systems proof

This is a public, non-confidential portfolio proof for Thomas Cloarec.

It publishes a small static React and TypeScript style proof that shows a mock evaluation and trace-review workflow using synthetic data only.

## Safety scope

This proof does not use private APIs, customer data, secrets, live analytics, Arize, OpenRouter, KYC, signatures, or binding approvals. All run names, trace events, tool calls, reviewers, notes, scores, and queue items are invented for the demo.

## Contents

- `index.html`: public static portfolio page.
- `src/App.tsx`: static React style UI source with three states: evaluation run list, trace detail with tool calls, and human review queue.
- `src/mockData.ts`: synthetic evaluation, trace, tool-call, and review data.
- `docs/claim-control-checklist.md`: source intake, launch-note structure, reviewer checklist, and unsupported-claim removal table.
- `docs/claims-supported.md`: exact supported and unsupported claim boundaries.

## Proof value

The proof supports a narrow claim: Thomas can design a readable mock evaluation and trace-review workflow, pair it with claim controls, and separate synthetic demo evidence from production claims.

It does not support production AI observability, production React depth, OpenRouter usage, Arize Phoenix usage, customer adoption, employment authority, or legal assertions.

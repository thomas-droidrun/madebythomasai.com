# Claim-control checklist for workflow demos

Use this checklist before publishing developer-facing workflow content.

## Strong claims that need proof

- Security claims.
- Reliability or uptime claims.
- Compatibility claims.
- Scale or performance claims.
- Production readiness claims.
- Customer adoption or community claims.
- Cost savings or speed improvement claims.

## Labels

- Verified: direct source or test evidence supports the claim.
- Inferred: reasonable interpretation, but the draft should soften the wording.
- Remove: no source, no test, or outside the author's authority.

## Final pass

- Source links are attached.
- Placeholders are clearly marked.
- Expected output is realistic.
- Failure mode or rollback guidance is visible.
- Unresolved facts have an owner.
